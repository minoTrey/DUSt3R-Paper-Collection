Appendices
A     Model Details
Our model operates within a scene ROI box defined as [−40, 40] m × [−40, 40] m × [−1, 5.4] m in the ego coordinate frame.
  Implicit BEV Projection. The calibration-free camera-to-BEV projection module employs two sequential cross-attention
blocks for each encoder layer. Each cross-attention block operates between image patch tokens and a set of BEV latent queries.
The BEV query space is defined as a grid of resolution 32 × 32, resulting in 1024 BEV queries per layer. Each BEV query has a
dimension of 1024. The four layer-specific BEV representations are concatenated along the feature dimension and upsampled
using DPT [Ranftl et al., 2021] to produce the final BEV representation at resolution 256 × 256 with 256-dimensional features.
  Training Hyperparameters. We use balanced query sampling along LiDAR rays, dividing the free space segment [0, d) into
K = 5 bins for stratified sampling. The positive sampling thickness hyperparameter τ is set to 0.1 meters. During training, we
sample 150K positive and 150K negative query points per sample, where negative queries consist of 120K points from stratified
bins and 30K points from the symmetric interval [d − τ, d) near object surfaces.

B     Evaluation Details
B.1    Point Map Estimation
Point maps for our model are rendered from the implicit occupancy field by sampling points along each LiDAR ray with a step
size of 0.05 m. For baseline comparisons, point maps and depth maps from baseline methods were filtered by computing a
visibility mask using LiDAR rays projected into camera views via camera calibrations.

B.2    Occupancy Estimation
For occupancy voxel grid evaluation, we aggregate point-wise occupancy predictions of our model from 8 points sampled
uniformly within each voxel by taking the maximum occupancy probability.

five datasets (AV2, nuScenes, ONCE, NuPlan, and Waymo).
